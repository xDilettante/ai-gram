# ai-gram v0.6.0 Draft

`v0.6.0` is the planned minor release for the production-readiness work completed after `v0.5.0`. It focuses on practical bot-building helpers, production-style examples, safer public example logging, and maintainer discipline for future Telegram Bot API updates.

This document is a draft. Do not tag, publish, or create a GitHub Release from it without explicit maintainer approval.

The module is still pre-v1, so public APIs may continue to evolve before `v1.0`.

## Status

- Release type: planned regular public release.
- Planned module version: `v0.6.0`.
- Current preparation status: local release-candidate gates and direct public consumer smoke for `main` have passed.
- Telegram Bot API coverage: Bot API 10.0 remains the latest official release observed during the 2026-05-14 lightweight audit; no missing official method wrappers were found.
- Sensitive and state-changing live smoke remains manual-only.
- Code generation was not introduced.

## Highlights

- Added typed callback data helpers in the `callback` package:
  - compact namespace/action/ID/page/expiry encoding;
  - safe parsing and typed parse errors;
  - 64-byte Telegram `callback_data` limit checks;
  - confirm/cancel and pagination helpers;
  - inline keyboard button construction.
- Added dispatcher integration for parsed callback actions:
  - `dispatch.CallbackAction`;
  - `Dispatcher.OnCallbackAction`;
  - `Dispatcher.OnCallbackActionFunc`;
  - `dispatch.CallbackDataHandler`.
- Added error classification helpers on top of `errors.APIError` for:
  - Telegram API errors;
  - `retry_after` rate limits;
  - chat migrations;
  - forbidden and not-found responses;
  - network errors;
  - context cancellation and deadline errors.
- Added Telegram identity helpers for group/admin workflows:
  - `telegram.Actor`;
  - message, callback query, and update actor helpers;
  - anonymous admin detection;
  - reply target helpers.
- Added production-style examples:
  - `examples/07_inline_panel`;
  - `examples/08_retry_sender`;
  - `examples/09_group_admin`;
  - `examples/10_moderation_skeleton`;
  - `examples/11_transport_parity`.
- Updated public examples to mask numeric private IDs more consistently in logs.
- Added maintainer workflow documents for Bot API update audits and next release-candidate gates.
- Added a lightweight Bot API 10.0 freshness audit dated 2026-05-14.

## Compatibility Notes

No breaking pre-v1 rename is currently planned for this release.

New public helpers are additive. Existing `v0.5.0` public API names such as `aigram.Config`, `bot.Config`, and `telegram.LoginURL` remain the current names.

The new callback helper package does not replace raw callback data usage. Applications can continue to use exact callback strings, while newer code can use typed callback data and dispatcher action routes.

The error taxonomy helpers do not add hidden automatic retries. Applications remain responsible for retry policy.

The identity helpers do not implement moderation policy. They only expose practical user/chat actor and reply-target extraction.

## Examples

The new production-style examples are intended to show reusable application structure without turning ai-gram into a framework:

- `examples/07_inline_panel` demonstrates typed callback data, pagination, and confirm/cancel actions.
- `examples/08_retry_sender` demonstrates explicit retry decisions for rate limits, transient network failures, and context deadlines.
- `examples/09_group_admin` demonstrates read-only group/admin identity commands with anonymous-admin and reply-target handling.
- `examples/10_moderation_skeleton` demonstrates dry-run moderation reports and admin previews without destructive Bot API calls.
- `examples/11_transport_parity` demonstrates shared dispatcher, middleware, and handlers running through either long polling or webhook intake.

## Manual-Only Live Smoke Areas

The following areas must not be live-smoked automatically:

- payments, Stars, gifts, paid media, refunds, and subscription flows;
- Passport data;
- Business APIs and business account mutation;
- managed bot token/access methods;
- guest mode flows;
- reaction deletion;
- admin/destructive chat methods;
- sticker set mutation;
- games, inline mode, Web App, Mini App, and prepared inline flows;
- lifecycle methods such as `logOut` and `close`;
- webhook certificate upload and webhook state changes.
