# ai-gram v0.5.0

`v0.5.0` is the regular public release for the post-`v0.4.0` cleanup on `main`. It packages the current pre-v1 API names, public consumer smoke tooling, and maintainer safety improvements into a normal GitHub Release and Go module tag.

This is not marked as a GitHub pre-release. The module is still pre-v1, so public APIs may continue to evolve before `v1.0`.

## Status

- Release type: regular public release.
- Module version: `v0.5.0`.
- Telegram Bot API coverage: code coverage for Bot API 10.0 is complete with documented architecture differences.
- Sensitive and state-changing live smoke remains manual-only.
- Public APIs may still evolve before `v1.0`.

## Highlights

- Renamed `bot.BotConfig` and root `aigram.BotConfig` to the idiomatic `Config`.
- Renamed `telegram.LoginUrl` to `LoginURL`.
- Added missing GoDoc for inline result JSON encoders and chat-member helper methods.
- Added `docs/PRE_V1_NOTES.md` as the current public API shape reference.
- Marked older Bot API 9.6 workstream documents as historical where later cleanup superseded compatibility notes.
- Added `scripts/smoke_public_consumer.sh` for repeatable external-module consumer smoke.
- Added the manually triggered `Public consumer smoke` GitHub Actions workflow.
- Hardened maintainer smoke tooling with dry-run support and broader identifier sanitization.

## Migration notes

This release contains breaking pre-v1 API cleanup:

- Replace `aigram.BotConfig` with `aigram.Config`.
- Replace `bot.BotConfig` with `bot.Config`.
- Replace `telegram.LoginUrl` with `telegram.LoginURL`.

The root package remains a compact quick-start facade. Use `github.com/xDilettante/ai-gram/bot` for the full outgoing Bot API method surface and `github.com/xDilettante/ai-gram/telegram` for Telegram data contracts.

## Install

Install the release with:


## Manual-only live smoke areas

The following areas must not be live-smoked automatically:

- payments, Stars, gifts, and subscription flows;
- Passport data;
- Business API state changes;
- managed bot token/access methods;
- guest mode flows;
- reaction deletion;
- admin/destructive chat methods;
- live-photo and poll-media checks that send visible media/polls;
- sticker set mutation;
- games requiring BotFather setup;
- inline mode flows requiring BotFather/client setup;
- lifecycle methods such as `logOut` and `close`;
- `setWebhook` certificate upload and webhook state changes.
