# ai-gram v0.3.0

`v0.3.0` is the first large pre-v1 release planned for the new public `ai-gram` repository. It packages the Bot API 9.6 coverage workstream, public repository polish, CI, MIT licensing, and repository maturity assets into one coherent beta milestone.

This release is prepared locally in advance of tagging. Do not create or publish the `v0.3.0` tag until the maintainer explicitly approves the release action.

## Status

- Release type: pre-v1 / beta.
- Telegram Bot API coverage: code coverage for Bot API 9.6 is complete with documented architecture differences.
- Official method wrappers: 169/169.
- Sensitive and state-changing live smoke remains manual-only.
- Public APIs may still evolve before v1.0.

## Architecture differences

The release keeps several intentional compatibility-oriented differences from the exact wording of the official Telegram Bot API documentation:

- `FileRef` / `FileUpload` instead of a direct official `InputFile` type.
- `GetChat` plus `GetChatFullInfo` compatibility split.
- A flat `ChatMember` compatibility strategy instead of exposing only concrete `ChatMember*` variants.
- `CallbackQuery.Message` plus `MaybeMessage` compatibility strategy for accessible and inaccessible callback messages.

These are documented design choices, not current coverage blockers.

## Manual-only live smoke areas

The following areas must not be live-smoked automatically:

- payments, Stars, gifts, and subscription flows;
- Passport data;
- Business API state changes;
- managed bot token methods;
- admin/destructive chat methods;
- sticker set mutation;
- games requiring BotFather setup;
- inline mode flows requiring BotFather/client setup;
- lifecycle methods such as `logOut` and `close`;
- `setWebhook` certificate upload and webhook state changes.

## Release checklist

When the maintainer explicitly approves publishing `v0.3.0`:

1. Create a local annotated tag:

   ```bash
   git tag -a v0.3.0 -m "Release v0.3.0"
   ```

2. Push only that tag:

   ```bash
   git push origin v0.3.0
   ```

3. Verify public module installation from the tag:

   ```bash
   go get github.com/xDilettante/ai-gram@v0.3.0
   ```

4. Create the GitHub Release using this document as the release notes source.

Do not run `git push --tags`, do not create unrelated tags, and do not create a GitHub Release before explicit approval.
