# ai-gram v0.1.1

Patch release that makes `ai-gram` installable as a canonical public Go module.

`v0.1.1` keeps the `v0.1.0` feature set and fixes the module path so external Go projects can use `go get github.com/xDilettante/ai-gram@v0.1.1`.

## What changed since v0.1.0

- Fixed Go module path:

  ```text
  module github.com/xDilettante/ai-gram
  ```

- Updated all internal project imports from `ai-gram/...` to `github.com/xDilettante/ai-gram/...`.
- Verified external consumer install:

  ```bash
  go get github.com/xDilettante/ai-gram@v0.1.1
  ```

- `v0.1.0` remains a historical milestone tag and should not be used as the canonical public install target.

## Included from v0.1.0 milestone

`v0.1.1` includes the first `ai-gram` milestone feature set:

- typed Bot API core with `BotConfig`, typed `ChatID`, private token storage, typed API errors, JSON calls, and multipart upload;
- webhook and long polling transports;
- send text/media methods for message, photo, document, video, audio, and voice;
- reply markup, callback query answer, edit text/caption/reply markup, delete message, reply parameters, and message thread IDs;
- forward/copy methods;
- chat actions and pin/unpin methods;
- chat/member info methods;
- moderation methods for ban, unban, and restrict;
- dispatcher, middleware chain, recovery/timeout/observe helpers, and access control middleware;
- admin-only examples and deploy/manual smoke harness with auto-discovery, auto SSH tunnel, Telegram notifications, deep-link smoke panels, safe logs, and release/smoke docs.

## Installation

Use the canonical module path:

```bash
go get github.com/xDilettante/ai-gram@v0.1.1
```

Minimal example with the current public API:

```go
package main

import (
    "context"
    "fmt"
    "log"

    aigram "github.com/xDilettante/ai-gram"
)

func main() {
    b, err := aigram.NewBot(aigram.BotConfig{Token: "123456:TEST_TOKEN"})
    if err != nil {
        log.Fatal(err)
    }

    // Do not print real bot tokens. Use GetMe for identity in real code.
    me, err := b.GetMe(context.Background())
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println(me.Username)
}
```

For a no-network compile smoke, create the bot and print only its type:

```go
b, err := aigram.NewBot(aigram.BotConfig{Token: "123456:TEST_TOKEN"})
if err != nil {
    log.Fatal(err)
}
fmt.Printf("%T\n", b)
```

## Verification

Before `v0.1.1`, the repository passed:

- `go test ./...`;
- `go vet ./...`;
- `go list ./...`;
- external temporary project check with `go get github.com/xDilettante/ai-gram@v0.1.1`, `go mod tidy`, `go run .`, and `go list -m all`.

Safe live smoke was completed before the `v0.1.0` milestone and the `v0.1.1` module-path patch, including local Bot API smoke, webhook deploy, access panel, edit/delete/caption/reply/forward/copy/chat-action/chat-info flows, and long polling auto-exit.

## Known limitations

- `ai-gram` does not yet provide full Telegram Bot API coverage.
- Remaining send methods, polls, stickers, invite links, join requests, inline mode, payments, Passport, Games, WebApp/LoginUrl, business APIs, Stars/gifts, and code generation are still planned or deferred.
- Admin/destructive methods require explicit care, suitable bot permissions, and dedicated test chats; they are not auto-smoked.
- Some Telegram types are intentionally minimal and should expand alongside implemented methods.
