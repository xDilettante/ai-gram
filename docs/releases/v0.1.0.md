# ai-gram v0.1.0

`ai-gram` is a Go Telegram Bot API library developed with an AI-assisted workflow: user as product/architecture lead, Web GPT and Codex GPT implementing and verifying the library.

This first milestone delivers a practical typed core, selected Bot API methods, update transports, dispatcher/middleware, examples, and a safe manual/live smoke contour. It is not full Telegram Bot API coverage yet.

## Highlights

- Typed Bot API client with token-safe configuration and error handling.
- Long polling runner and inbound webhook handler.
- Dispatcher with route predicates and middleware chain.
- Admin-only access control middleware used by examples by default.
- Text/media send methods, reply parameters, message thread IDs, callback/edit/delete flows, forward/copy, chat actions, pin/unpin, chat/member info, and moderation methods.
- File upload/download helpers without exposing token-bearing download URLs.
- Manual smoke/deploy harness with auto-discovery, role-specific tokens, temporary SSH tunnel support, Telegram notifications, deep-link smoke panels, and safe logs.

## Implemented Bot API areas

- Core identity and errors: `GetMe`, typed `APIError`, response parameters.
- Updates/webhooks: `GetUpdates`, `SetWebhook`, `DeleteWebhook`, `GetWebhookInfo`, long polling runner, inbound webhook handler.
- Send/media/files: `SendMessage`, `SendPhoto`, `SendDocument`, `SendVideo`, `SendAudio`, `SendVoice`, `GetFile`, `DownloadFile`, `FileID`, `FileURL`, `FileUpload`.
- Reply/callback/edit/delete: reply markup types, `ReplyParameters`, `AnswerCallbackQuery`, `EditMessageText`, `EditMessageCaption`, `EditMessageReplyMarkup`, `DeleteMessage`.
- Forward/copy: `ForwardMessage`, `CopyMessage`.
- Chat actions/pin: `SendChatAction`, `PinChatMessage`, `UnpinChatMessage`, `UnpinAllChatMessages`.
- Chat/member info: `GetChat`, `GetChatMember`, `GetChatAdministrators`, `GetChatMemberCount`.
- Moderation: `BanChatMember`, `UnbanChatMember`, `RestrictChatMember`.

See [`docs/API_COVERAGE.md`](../API_COVERAGE.md) for the detailed method map and risk classification.

## Live verification

Safe live verification was completed before publishing the tag:

- local Bot API smoke;
- webhook deploy;
- access panel/status/open/close;
- access denied behavior;
- edit text and reply markup;
- generated document caption edit;
- delete message limited to bot-created test messages;
- reply parameters;
- send chat action;
- forward/copy;
- getChat info;
- long polling smoke with auto-exit after successful update and reply.

Destructive/admin checks such as ban/restrict, logOut, close, migration, and broad admin operations were not run automatically.

## Security and safety

- Raw bot token access through public `Bot.Token()` was removed before `v0.1.0`.
- Errors, scripts, deploy logs, notifications, and reports are designed to avoid token leakage.
- `.env.local`, `.deploy/generated.env`, keys, and generated deploy secrets are ignored.
- Examples default to admin-only access control.
- Safe logs include operational IDs such as update/action/chat/message IDs, but not full private message text.

## Known limitations

- The project does not implement full Telegram Bot API coverage.
- Remaining send methods such as contact/location/venue/poll/dice/sticker/animation/video note/media group are deferred.
- Invite links, join requests, promote/admin management, inline mode, payments, Passport, Games, WebApp/LoginUrl, business APIs, Stars/gifts, and full code generation are deferred.
- Some Telegram types are intentionally minimal and should grow alongside implemented methods.
- Admin/destructive methods require dedicated test chats and explicit operator intent.
- At the `v0.1.0` tag, the module path was `ai-gram`; public Go module import-path stabilization is prepared for the next patch release.

## Installation

At the `v0.1.0` tag, the module path in `go.mod` was:

```text
module ai-gram
```

For this milestone, use the published tag as a source checkpoint:

```bash
git clone git@github.com:xDilettante/ai-gram.git
cd ai-gram
git checkout v0.1.0
go test ./...
```

Because the module path at this tag was `ai-gram`, a direct public `go get github.com/xDilettante/ai-gram@v0.1.0` is not the canonical install command for this tag. The canonical public module path is prepared for the next patch release.
