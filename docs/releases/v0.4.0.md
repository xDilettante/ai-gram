# ai-gram v0.4.0

`v0.4.0` is the pre-v1 release after the published `v0.3.0` tag. It packages the Telegram Bot API 10.0 coverage workstream and the follow-up release-readiness documentation.

This release was published after explicit maintainer approval as an annotated tag and GitHub pre-release.

## Status

- Release type: pre-v1 / beta.
- Telegram Bot API coverage: code coverage for Bot API 10.0 is complete with documented architecture differences.
- Official Bot API 9.6 method wrappers: 169/169; Bot API 10.0 additions are implemented and audited.
- Sensitive and state-changing live smoke remains manual-only.
- Public APIs may still evolve before v1.0.

## Highlights

- Completed Telegram Bot API 10.0 code coverage on `main`.
- Completed the Bot API 10.0 additions released on May 8, 2026.
- Added Bot API 10.0 guest mode, reaction deletion, poll media, live photos, managed bot access settings, personal chat message reads, and empty-text message drafts.
- Kept the Bot API 9.6 `setWebhook.certificate` upload fix and the 169/169 official method wrapper coverage from `v0.3.0`.
- Updated maintainer release readiness, live-smoke planning, roadmap, and coverage documentation for Bot API 10.0.

## Architecture notes

The release keeps a small set of intentional Go API design choices:

- `FileRef` / `FileUpload` instead of a direct official `InputFile` type.
- `GetChatFullInfo` remains as a same-result pre-v1 alias for `GetChat`; `GetChat` returns `ChatFullInfo`.
- `telegram.ChatMember` is an interface implemented by official `ChatMember*` variants.
- `CallbackQuery.Message` uses the official `MaybeInaccessibleMessage` shape.
- Live-photo URL inputs are intentionally rejected in places where the implementation requires `file_id` or multipart upload for the official live-photo payload.

These are documented design choices, not current coverage blockers.

## Manual-only live smoke areas

The following areas must not be live-smoked automatically:

- payments, Stars, gifts, and subscription flows;
- Passport data;
- Business API state changes;
- managed bot token/access methods;
- guest mode flows;
- reaction deletion;
- admin/destructive chat methods;
- live-photo and poll-media checks that send visible media/polls;
- sticker set mutation;
- games requiring BotFather setup;
- inline mode flows requiring BotFather/client setup;
- lifecycle methods such as `logOut` and `close`;
- `setWebhook` certificate upload and webhook state changes.
